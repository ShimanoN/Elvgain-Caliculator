# Elevation Loom ユーザーマニュアル

## はじめに

Elevation Loom（標高獲得トラッカー）へようこそ！このアプリケーションは、週間の標高獲得目標を設定し、日々のトレーニング進捗を記録・管理するためのツールです。

## アクセス方法

### ローカル環境
```bash
cd /path/to/Elvgain-Caliculator
npm run dev
```
ブラウザで `http://localhost:8000` にアクセスします。

### 本番環境
https://elevation-loom.web.app

## 主な機能

### 1. 日次データ入力（メインページ）

#### 基本的な使い方
1. メインページ（index.html）を開きます
2. 「標高獲得 Part 1」と「Part 2」に数値を入力します
   - Part 1: 午前のトレーニング
   - Part 2: 午後のトレーニング
3. 入力後、他の場所をクリックすると自動保存されます
4. 日次合計が自動計算されて表示されます

#### 日付の移動
- **前日ボタン**: 前日のデータを表示・編集
- **翌日ボタン**: 翌日のデータを表示・編集
- **今日ボタン**: 本日のデータに戻る

#### 表示情報
- **現在の日付**: 編集中の日付が表示されます
- **週の範囲**: 現在の週の期間（月曜日〜日曜日）
- **週合計**: 今週の標高獲得合計
- **残り日数**: 週の残り日数

### 2. 週目標設定（週目標ページ）

#### 目標の設定
1. メインページから「週目標を見る」リンクをクリック
2. 「週目標」欄に目標値を入力（例: 3000m）
3. 自動保存されます

#### 週スケジュール
- 月曜日から日曜日までの7日間が表示されます
- 各日の標高獲得が表示されます
- 実績がある日は背景色が変わります

#### 進捗チャート
- 週の進捗が視覚的に表示されます
- 目標に対する達成率がパーセンテージで表示されます

### 3. データのバックアップ

#### 手動バックアップ
ブラウザのコンソール（F12 → Console）で以下を実行：
```javascript
// バックアップ作成
const backup = window.elvBackup.exportBackup();
console.log(JSON.stringify(backup));
// コピーしてテキストファイルに保存

// リストア
const backupData = { /* バックアップデータをペースト */ };
window.elvBackup.importBackup(backupData);
```

#### 自動バックアップ
- データ保存時に自動的にLocalStorageにバックアップされます
- 定期的にバックアップを手動で保存することをおすすめします

### 4. チャートのエクスポート

週目標ページで進捗チャートを画像として保存できます：
1. 週目標ページを開く
2. エクスポートボタンをクリック
3. PNG画像としてダウンロードされます

## データ入力のベストプラクティス

### 毎日の習慣
1. **トレーニング後すぐに入力**: データを忘れないうちに記録
2. **週の初めに目標設定**: 月曜日に週目標を設定
3. **週末に振り返り**: 週目標ページで達成状況を確認

### 30日間連続データ入力チャレンジ

Phase 6の目標として、30日間連続でデータを入力します。

#### チェックリスト
- [ ] 1週目（1-7日目）: 毎日データ入力
- [ ] 2週目（8-14日目）: 毎日データ入力
- [ ] 3週目（15-21日目）: 毎日データ入力
- [ ] 4週目（22-28日目）: 毎日データ入力
- [ ] 5週目（29-30日目）: 毎日データ入力

#### 記録のコツ
1. **リマインダー設定**: スマートフォンのアラームで入力時間を設定
2. **ルーティン化**: トレーニング後の決まった時間に入力
3. **週次レビュー**: 日曜日の夜に週のデータを確認

## トラブルシューティング

### データが保存されない場合
1. ブラウザのJavaScriptが有効になっているか確認
2. プライベートブラウジングモードではないか確認
3. ブラウザのストレージ容量を確認
4. F12でコンソールを開き、エラーメッセージを確認

### データが消えた場合
1. バックアップからリストアを試す
2. ブラウザの履歴からデータを確認
3. Firebase Consoleでデータを確認（管理者のみ）

### パフォーマンスが遅い場合
1. ブラウザのキャッシュをクリア
2. 他のタブを閉じる
3. ブラウザを再起動

## データのプライバシー

- データはFirebase Firestoreに保存されます
- ユーザー認証が有効な場合、自分のデータのみアクセス可能
- データは暗号化されて転送されます
